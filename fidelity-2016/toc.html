9.00-10.00 Welcome, Overview, and Motivation: APIs increasingly become the connective tissue of IT in general, and the main way of how business is looking at their IT strategy and structure in particular. APIs are what both separate and connect collaborating and communicating peers: They facilitate loose coupling of service providers and consumers and thus strike a balance between the ability to collaborate and cooperate, while still maintaining independence and individual control. Welcome, Overview, and Motivation (28 Slides) Competitive Advantage [http://www.hbs.edu/faculty/Pages/item.aspx?num=193] · Unbundling the Corporation [https://hbr.org/1999/03/unbundling-the-corporation] · Digitally Remastered [https://www.ca.com/us/collateral/ebook/building-software-into-your-business-dna] · Software Architecture [https://leanpub.com/software-architecture-for-developers]
10.00-11.00 Introduction to API Design: API Design is about loose coupling, making sure that every service is entirely self-contained and can only be interacted with through its API. Another important aspect of this architectural approach is that designers and developers have a lot of autonomy when it comes to designing, developing, and deploying their services. For this reason, API design for services and microservices must specifically target the diversity and decentralization that are inherent properties of any (micro)services landscape. Introduction to API Design (26 Slides) RESTful Cookbook [http://shop.oreilly.com/product/9780596801694.do] · RESTful APIs [http://shop.oreilly.com/product/0636920028468.do]
11.00-12.00 API Design Advanced Topics: Microservice Architecture and Microservices embrace a new style of IT development and culture that focuses on speed, safety, robustness, and scalability. Taking the ideas of agile development, CI/CD, and DevOps one step further, microservices embrace organizational changes as well, so that improved IT processes are not compromised by the side-effects of organizational realities. API Design Advanced Topics (22 Slides) Microservices Book [http://transform.ca.com/API-microservice-architecture-oreilly-book.html] · Fowler on Microservices [http://www.martinfowler.com/articles/microservices.html] · Loose Coupling [http://dret.net/netdret/publications#pau09a]
13.00-14.30 API Management: API management needs to work both on a technical as well as on an organizational/cultural level. For the technical level, The most challenging factor in large API landscapes are the number of services, the rate of change, and the diversity of API designs in that landscape. API management needs to be able to deal with these factors in ways which do not inhibit the organic growth of the API ecosystem, while still maintaining a level of oversight and monitoring that make it possible to support data-driven decision-making. API Management (34 Slides) API Stylebook [http://apistylebook.com/] · IETF [http://ietf.org/] · W3C [http://www.w3.org/] · Web Concepts [http://webconcepts.info/]
14.30-15.15 API Evolution: API landscapes are constantly evolving: New APIs are added, existing APIs change, and established APIs are deprecated and eventually retired. API Evolution requires to have a set of practices that support all these activities. These practices should be easy to follow, should be visible through the APIs themselves, and thus allow the landscape to make its own evolution visible and manageable. API Evolution (23 Slides) Robust Extensibility [http://dret.typepad.com/dretblog/2016/04/robust-extensibility.html]
15.15-16.00 API Security: APIs are the visible and accessible surface of an organization's IT. This presentation presents different types of attacks, an introduction to security fundamentals and technologies, and how these apply to APIs.
