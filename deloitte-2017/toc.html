API Styles: APIs as a medium for communications can follow different styles. Knowing the styles, picking the ones that are a good choice for a given scenario, and establishing an internal API stylebook are important aspects of an API strategy. API Styles (26 Slides) Loose Coupling [http://dret.net/netdret/publications#pau09a] · API Stylebook [http://apistylebook.com/]
Change Management and Versioning: Like everything on a long-enough timeline, APIs do change over time. Having a plan for how change is managed, and encouraging API designs that are better at dealing with changes can make a big difference for the effectiveness of an API landscape. Change Management and Versioning (18 Slides) RFC 6648 [https://tools.ietf.org/html/rfc6648] · Robust Extensibility [http://dret.typepad.com/dretblog/2016/04/robust-extensibility.html] · Change Management [https://blog.goodapi.co/api-change-management-2fe5bba32e9b] · Semantic Versioning [http://semver.org/]
Introduction to Microservices: APIs are interfaces to capabilities, which then must be implemented in some shape or form. As a less heavyweight alternative to more traditional integrated systems, microservices take an "ecosystem approach", encouraging organizations to make sure that adding, changing, or removing capabilities is something that they can do more easily. Introduction to Microservices (27 Slides) MSA Book [http://www.apiacademy.co/microservice-architecture-the-oreilly-book/] · Microservices Resource Guide [https://martinfowler.com/microservices/]
